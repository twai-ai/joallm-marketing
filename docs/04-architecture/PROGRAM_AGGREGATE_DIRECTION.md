## The real primary object

ATRISI is not a marketing platform. Marketing is not even the primary product lens.

The durable business object is **Program**.  
The institutional philosophy is **Institution Capability**.

```text
Institution
      │
      ▼
Program
      │
      ├── Growth          ← formerly “Marketing” / Program Growth workspace
      ├── Admissions
      ├── Learning
      ├── Assessment
      ├── Placement
      └── Alumni
```

**Growth** is one workspace around a Program — peer to Admissions and Learning — not a separate product island.

Public ATRISI catalog today: [atrisi.org/programs](https://atrisi.org/programs)  
(Resonance · Amplify · Campus-to-Career · AI Native Systems Engineering · Resilient Systems)

---

## Why this differentiates ATRISI

| Others start with | ATRISI starts with |
|---|---|
| Campaign | **Program** |
| Leads | Applications → Admissions → Enrollment |
| Channel vanity metrics | Program outcomes (enrollment quality, placement, learning) |

Campaigns, creatives, and channels become **contextual**. Creative AI gets richer context. Admissions is a natural continuation. Learning completes the lifecycle.

This fits education, skilling, executive education, incubators, accelerators, CSR initiatives, and research programs.

## Brain vs Workspace

| Layer | Scope | Questions |
|---|---|---|
| **Brain** (ATRISI Marketing / cross-program IQ) | Organizational | Which Programs grow fastest? Which creatives win across institutions? Which channels fit exec ed vs undergrad? |
| **Program Workspace** | Single Program | Run Growth / Admissions / Learning for *this* Program |

Brain = cross-program intelligence.  
Workspace = program-specific operating system surface.

## Program aggregate (define this next)

```text
Program
├── Identity          name, family, institution, status
├── Positioning       tagline, value props, differentiation
├── Audience          segments, personas, ICPs
├── Pricing           tuition, scholarships, packages
├── Cohorts           intakes, dates, capacity
├── Growth            campaigns, creatives, channels, applications
├── Admissions        pipeline, decisions, offers
├── Delivery          Learning · Assessment (Education Brain)
├── Outcomes          placement, alumni, capability evidence
└── Intelligence      program-scoped analytics + Brain rollups
```

Canonical Programs are seeded from the public catalog; institutions later add their own Programs (MBA, BTech, …) and each automatically receives Growth · Admissions · Learning · Assessment · Outcomes — no separate suite setup.
